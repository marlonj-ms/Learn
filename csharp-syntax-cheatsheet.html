1. 变量声明与 var

// 显式类型
int    age   = 30;
string name  = "Alice";
bool   isOn  = true;

// 用 var 让编译器推断类型（需要等号右边能推断出类型）
var age2  = 30;          // 推断为 int
var name2 = "Alice";     // 推断为 string
var list  = new List<int>();   // 推断为 List<int>
var 不能用的场景：
  • 没有初始化：var x; ❌
  • 初始化为 null：var x = null; ❌
  • 用来装 lambda：var f = n => n + 1; ❌（lambda 自己没有类型）
  • 字段或方法参数（var 只能用在方法内的局部变量）

2. 数组（Array）的多种定义

// 方式 1：显式 new（最完整写法）
int[] a = new int[] { 1, 2, 3 };

// 方式 2：省略 new，仅在变量声明时合法
int[] b = { 1, 2, 3 };

// 方式 3：collection expression（C# 12，最简洁）
int[] c = [1, 2, 3];

// 方式 4：先指定大小，后赋值
int[] d = new int[5];      // [0, 0, 0, 0, 0]
d[0] = 10;
d[1] = 20;

// 方式 5：用 var
var e = new int[] { 1, 2, 3 };    // ✅ 推断 int[]
var f = new[] { 1, 2, 3 };        // ✅ 推断 int[]（元素类型从值推断）
var g = { 1, 2, 3 };              // ❌ 不行，{...} 不是表达式

多维数组

// 二维数组
int[,] grid = new int[3, 4];        // 3 行 4 列
int[,] m = { { 1, 2 }, { 3, 4 } };  // 2x2

// 数组的数组（jagged array）
int[][] jagged = new int[3][];
jagged[0] = new int[] { 1, 2 };
jagged[1] = new int[] { 3, 4, 5 };

字符串数组的常见写法

string[] names1 = new string[] { "Alice", "Bob", "Charlie" };
string[] names2 = { "Alice", "Bob", "Charlie" };
string[] names3 = ["Alice", "Bob", "Charlie"];
常见错误：
  • 把 'a'（char）当 "a"（string）写：单引号是字符，双引号是字符串
  • 把 { 1, 2, 3 } 直接传给方法 —— 这种字面量只能用在变量声明右侧

3. List<T> 的定义与初始化

// 方式 1：空 List
List<int> a = new List<int>();

// 方式 2：target-typed new（C# 9，最常用）
List<int> b = new();

// 方式 3：带初始元素（集合初始化器）
List<int> c = new() { 1, 2, 3 };

// 方式 4：从另一个集合创建
List<int> d = new(c);          // 复制 c
List<int> e = new List<int>(c);

// 方式 5：collection expression（C# 12）
List<int> f = [1, 2, 3];

常用操作

List<string> names = new() { "Alice", "Bob" };

names.Add("Charlie");           // 追加
names.AddRange(new[] { "D", "E" });  // 追加多个
names.Insert(0, "First");       // 在索引 0 处插入
names.Remove("Bob");            // 删除（第一个匹配）
names.RemoveAt(0);              // 按索引删除
names.Count;                    // 元素个数
names[0];                       // 索引访问
names.Contains("Alice");        // 是否存在
names.Sort();                   // 原地排序

4. Dictionary<K,V> 的定义与初始化

// 方式 1：空字典
Dictionary<string, int> a = new Dictionary<string, int>();

// 方式 2：target-typed new
Dictionary<string, int> b = new();

// 方式 3：集合初始化器（用 { key, value } 形式）
Dictionary<string, int> c = new()
{
    { "alice", 30 },
    { "bob", 25 },
};

// 方式 4：索引器初始化器（推荐，更直观）
Dictionary<string, int> d = new()
{
    ["alice"] = 30,
    ["bob"]   = 25,
};

常用操作

Dictionary<string, int> ages = new()
{
    ["alice"] = 30,
    ["bob"]   = 25,
};

ages["charlie"] = 40;            // 添加或更新
ages.Add("david", 50);           // 添加（key 已存在则抛异常）
ages.Remove("bob");              // 删除
ages.ContainsKey("alice");       // 检查 key
ages["alice"];                   // 取值（key 不存在抛异常）

// 安全取值
if (ages.TryGetValue("alice", out int age))
{
    Console.WriteLine(age);
}

// 遍历
foreach (var kvp in ages)
{
    Console.WriteLine($"{kvp.Key} = {kvp.Value}");
}

5. target-typed new()（C# 9）

当左边已经有类型时，右边可以省略类型名：

// 完整写法
Dictionary<string, List<int>> map = new Dictionary<string, List<int>>();

// target-typed new
Dictionary<string, List<int>> map = new();

// 任何类型都适用
List<int>       a = new();
StringBuilder   b = new();
Person          c = new();
HashSet<string> d = new();

// 带参数也可以
StringBuilder e = new(100);     // 初始容量 100
Person f = new("Alice", 30);    // 调用构造函数
记忆口诀： new() 意思是 "new 一个左边那个类型"。

7. 对象初始化器（Object Initializer）

创建对象后立即给属性赋值，不用调用构造函数也能初始化：

public class Person
{
    public string Name { get; set; }
    public int    Age  { get; set; }
}

// 传统方式
Person p1 = new Person();
p1.Name = "Alice";
p1.Age  = 30;

// 对象初始化器
Person p2 = new Person { Name = "Alice", Age = 30 };

// target-typed new + 对象初始化器
Person p3 = new() { Name = "Alice", Age = 30 };

// 集合 + 对象初始化器组合
List<Person> people = new()
{
    new() { Name = "Alice", Age = 30 },
    new() { Name = "Bob",   Age = 25 },
};

8. Lambda 表达式的各种形态

// 无参数
Action a = () => Console.WriteLine("hi");

// 一个参数（括号可省略）
Func<int, int> b = n => n * 2;
Func<int, int> c = (n) => n * 2;       // 等价

// 多个参数（括号必须）
Func<int, int, int> d = (a, b) => a + b;

// 显式类型（少用）
Func<int, int> e = (int n) => n * 2;

// 语句体（statement lambda）：多行逻辑用 { }
Func<int, int> f = n =>
{
    Console.WriteLine($"got {n}");
    return n * 2;
};

// 无返回值的 lambda（用于 Action）
Action<string> g = msg => Console.WriteLine(msg);
Action<string> h = msg =>
{
    Console.WriteLine("logging:");
    Console.WriteLine(msg);
};
表达式体 vs 语句体：
  • n => n * 2 = 表达式体（expression lambda），直接返回表达式的值
  • n => { return n * 2; } = 语句体（statement lambda），必须用 return
↑ 回到顶部

9. 字符串：插值、$、@、"""

string name = "Alice";
int age = 30;

// 拼接（最原始）
string s1 = "Name: " + name + ", Age: " + age;

// 插值字符串（推荐）
string s2 = $"Name: {name}, Age: {age}";

// 表达式可以直接放在 {}
string s3 = $"Total: {age * 2}";
string s4 = $"Upper: {name.ToUpper()}";

// 格式化
string s5 = $"Price: {123.456:F2}";   // F2 = 保留 2 位小数

// @ 字符串（不处理转义符）
string path1 = "C:\\Users\\Marlon";
string path2 = @"C:\Users\Marlon";    // 等价但更易读

// $@ 或 @$ 组合（插值 + 不转义）
string s6 = $@"Path: C:\Users\{name}";

// 多行字符串（C# 11+，用 """）
string s7 = """
    {
        "name": "Alice",
        "age": 30
    }
    """;

10. PascalCase / camelCase 命名约定

类型约定示例
类名（Class）PascalCasePerson, OrderService
方法（Method）PascalCaseGetName(), PlaceOrder()
属性（Property）PascalCaseName, OrderId
公共字段（public field）PascalCasepublic int Count;
常量（const）PascalCaseconst int MaxSize = 100;
局部变量（local var）camelCaseint orderCount
方法参数camelCasevoid M(int orderId)
私有字段（private field）_camelCaseprivate int _count;
接口I + PascalCaseIList, IDisposable
常见错误：
  • .where() ❌ → .Where() ✅（LINQ 方法都是 Pascal）
  • .length ❌ → .Length ✅（属性都是 Pascal）
  • list.add() ❌ → list.Add() ✅

11. 方法签名（method signature）

// 完整方法定义
public int Add(int a, int b)
{
    return a + b;
}
// ↑         ↑   ↑          ↑    ↑
// 访问修饰符 返回值 方法名     参数  方法体

// 表达式体方法（expression-bodied method，C# 6+）
public int Add(int a, int b) => a + b;

// 无返回值
public void PrintHello()
{
    Console.WriteLine("Hello");
}

// 静态方法（不需要实例）
public static int Square(int n) => n * n;

// 带默认参数
public void Greet(string name = "Guest")
{
    Console.WriteLine($"Hello, {name}");
}

// 命名参数调用
Greet(name: "Alice");

// 可变参数（params）
public int Sum(params int[] numbers) => numbers.Sum();
Sum(1, 2, 3, 4);   // 任意个数

13. 条件运算符与 null 检查

// 三元运算符
int abs = n < 0 ? -n : n;

// null 合并（??）
string name = userName ?? "Guest";    // userName 为 null 时用 "Guest"

// null 合并赋值（??=）
list ??= new List<int>();             // list 为 null 时才赋值

// null 条件访问（?.）
int? len = name?.Length;              // name 为 null 时返回 null，不会崩

// 链式 null 条件
int? len2 = person?.Address?.Street?.Length;

// 与 ?? 配合
int len3 = name?.Length ?? 0;         // null 时返回 0

// pattern matching
if (obj is string s)
{
    Console.WriteLine(s.Length);      // s 已经是 string 类型
}

if (obj is null) { ... }
if (obj is not null) { ... }

14. 类型转换（cast / convert）

// 隐式转换（安全，编译器自动）
int   i = 10;
long  l = i;          // int -> long，OK
double d = i;         // int -> double，OK

// 显式转换（cast，可能丢失精度或溢出）
double pi = 3.14;
int x = (int)pi;      // 3（小数被截断）

// 字符串 / 数字互转
string s1 = 123.ToString();           // int -> string
int    n1 = int.Parse("123");         // string -> int（失败抛异常）
bool   ok = int.TryParse("abc", out int n2);   // 安全转换

// Convert 类（更通用）
int x1 = Convert.ToInt32("123");
double y1 = Convert.ToDouble("3.14");

// 引用类型转换
object o = "hello";
string s2 = (string)o;       // 失败抛异常
string s3 = o as string;     // 失败返回 null
if (o is string s4) { ... }  // 安全检查

15. foreach 与 for 的选择

List<string> names = new() { "Alice", "Bob", "Charlie" };

// foreach：遍历所有元素，最常用
foreach (string name in names)
{
    Console.WriteLine(name);
}

// for：需要索引时
for (int i = 0; i < names.Count; i++)
{
    Console.WriteLine($"{i}: {names[i]}");
}

// while：需要复杂退出条件
int j = 0;
while (j < names.Count && names[j] != "Bob")
{
    j++;
}

// do-while：至少执行一次
do
{
    // ...
} while (condition);
选择规则：
  • 大部分情况用 foreach，简单不易错
  • 需要修改集合（边遍历边删除）时不要用 foreach，会抛异常
  • 需要索引、需要倒序、需要跳元素 → 用 for
  • 条件不固定 → 用 while

16. 扩展方法（Extension Methods）

public static class MyExtensions   // ① 必须 static class
{
    public static bool IsEven(this int n)  // ② static  ③ this 第一参数
        => n % 2 == 0;

    // 多参数：第一参数自动从"点前"对象拿，后面的显式传
    public static int AddBy(this int age, int y) => age + y;
}

// 两种调用完全等价：
5.IsEven();                  // 点语法（漂亮）
MyExtensions.IsEven(5);      // 静态调用（编译器实际生成的）

10.AddBy(3);                 // → 13（this=10, y=3）
三铁律：
  1. 类必须是 static class
  2. 方法必须是 static
  3. 第一参数前加 this —— 决定能扩展给哪种类型
查找规则：编译器靠 using 命名空间 + this 参数类型定位扩展方法，跟 class 叫什么名字无关。 所以 using System.Linq; 才让 list.Where(...) 可用。
↑ 回到顶部

=> 三种身份对照

符号位置含义例子
= 变量声明后 赋值 int x = 5;
=> 参数列表后 n => Lambda 箭头（分隔参数和函数体） n => n + 1
=> 方法/属性签名后 ) => 表达式体（替代 { return ... }） int Add(int n) => n + 1; 
// 经典：一行里两个 => 含义完全不同
public Func<int, int> MakeAdder(int n) => x => x + n;
//                                     ↑   ↑
//                          表达式体（return）  Lambda 箭头
识别口诀：
  • 左边是参数 → Lambda 箭头
  • 左边是方法/属性签名 → 表达式体语法糖
陷阱：表达式体 => 后面只能跟一个表达式，不能跟代码块 { ... }。 多行逻辑必须老老实实写 { }。

18. 集合长度三表（Length / Count / Count()）

类型怎么拿长度类型
int[]（数组） arr.Length 属性
string s.Length 属性
List<T> list.Count 属性
Dictionary<K,V> / HashSet<T> .Count 属性
IEnumerable<T> seq.Count() LINQ 扩展方法
陷阱：IEnumerable<T> 没有 .Length 也没有 .Count 属性！ 要么用 LINQ 的 .Count()（会遍历整个序列），要么自己 foreach 计数。 原因：IEnumerable 可能是无限序列，没有"长度"的概念。 
// ❌ 错误：IEnumerable 没有 Length
public int Average2(IEnumerable<int> nums)
{
    int sum = 0;
    foreach (int n in nums) sum += n;
    return sum / nums.Length;   // 编译错误！
}

// ✅ 正确：边遍历边计数
public int Average2(IEnumerable<int> nums)
{
    int sum = 0, count = 0;
    foreach (int n in nums) { sum += n; count++; }
    return sum / count;
}

// ✅ 也可以用 LINQ
using System.Linq;
int avg = nums.Sum() / nums.Count();
