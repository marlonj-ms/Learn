IList<T> 与泛型

2026-04-15 · 阅读时间约 5 分钟

1. 一句话总结

泛型让一个类/方法能"为任何类型工作"，接口让"使用者不关心具体实现"。
IList<T> 就是"任何能按索引访问的元素序列"的契约。

2. 栈 vs 堆（Stack vs Heap）

位置装什么例子
栈 Stack方法的局部变量、值类型本体int x = 5; 中的 5
堆 Heap引用类型的对象本体new List<int>() 创建的列表
 
int x = 5;                     // x 和 5 都在栈
string name = "Alice";         // name(引用)在栈，"Alice"对象在堆
List<int> list = new();        // list(引用)在栈，列表对象在堆

3. 值类型 vs 引用类型

值类型（value type）引用类型（reference type）
赋值时复制内容赋值时复制引用（指向同一对象）
int, bool, double, struct, enumclass, string, array, List, interface
 
// 值类型：复制
int a = 5;
int b = a;
b = 10;        // a 还是 5

// 引用类型：共享
List<int> x = new() { 1, 2 };
List<int> y = x;
y.Add(3);      // x 也变成 {1, 2, 3}！

4. 字符串不可变（string immutability）

string s = "hello";
s = s.ToUpper();        // s 现在指向新字符串 "HELLO"
                        // 原来的 "hello" 还在堆上，等待 GC
字符串的所有操作（ToUpper、Replace、Trim...）从不修改原字符串，而是返回新字符串。

5. 数组 vs List<T>

数组 int[]List List<int>
大小固定可动态增长
底层连续内存底层还是数组，满了就"翻倍 + 拷贝"
添加不能直接加.Add()

6. 泛型（generics）

<T> 是"类型占位符"，调用时由编译器替换为具体类型。

// 一份代码，多种类型复用
public class Box<T>
{
    public T Value;
}

Box<int>    a = new() { Value = 5 };
Box<string> b = new() { Value = "hi" };

// 编译器为每个 T 生成对应版本
// 比 object 更类型安全，比写多个 BoxInt/BoxString 更省代码

7. IList<T> 接口

接口是一份"必须实现的方法清单"，使用者不关心具体是谁实现的。

// IList<T> 大致约定：
//   .Count
//   .[index]            （索引访问）
//   .Add(item)
//   .Remove(item)
//   .Insert(index, item)
//   .Clear()

// List<T>、Array、ObservableCollection<T> 都实现了 IList<T>

// 函数参数声明成 IList<T>：
void Print(IList<string> items)
{
    for (int i = 0; i < items.Count; i++)
        Console.WriteLine(items[i]);
}

// 调用方可以传 List 也可以传数组
Print(new List<string> { "a", "b" });
Print(new string[]     { "a", "b" });

8. 嵌套泛型 IList<IList<string>>

// 外层 IList 装的元素是"IList<string>"
IList<IList<string>> matrix = new List<IList<string>>
{
    new List<string> { "a", "b" },
    new List<string> { "c", "d" },
};

matrix[0][1];   // "b"

9. 自测

问题答案
string 是值类型还是引用类型？引用类型（但行为像值类型 → 不可变）
List 满了之后怎么扩容？分配更大的内部数组（通常翻倍），把元素拷贝过去
<T> 是什么？类型占位符，调用时由编译器替换
为什么参数用 IList 而不是 List？调用方可以传任何实现了 IList 的类型，更灵活
赋值 List a = b 之后修改 a，b 会变吗？会，因为是引用类型，a 和 b 指向同一对象
