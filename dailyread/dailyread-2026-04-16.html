LINQ 基础五件套

2026-04-16 · 阅读时间约 5 分钟

1. 一句话总结

LINQ = 对集合做"过滤 / 转换 / 排序 / 分组 / 取值"的链式高阶方法。
心法：集合.操作方法(规则lambda)

2. 五个核心方法

List<int> numbers = new() { 1, 2, 3, 4, 5 };

// Where：过滤
var evens = numbers.Where(n => n % 2 == 0);        // {2, 4}

// Select：转换
var tens = numbers.Select(n => n * 10);            // {10, 20, 30, 40, 50}

// OrderBy / OrderByDescending：排序
var asc  = numbers.OrderBy(n => n);                 // 升序
var desc = numbers.OrderByDescending(n => n);       // 降序

// FirstOrDefault：取第一个（空集合返回默认值）
int first = numbers.First();                         // 1，空集合会崩
int safe  = numbers.FirstOrDefault();                // 1，空集合返回 0
string s  = names.FirstOrDefault(n => n.Length > 5); // 带条件

// GroupBy：分组
var grouped = names.GroupBy(n => n[0]);             // 按首字母分组
foreach (var g in grouped)
{
    Console.WriteLine($"Key: {g.Key}");
    foreach (var item in g) Console.WriteLine($"  {item}");
}

3. 形状要求

方法lambda 形状
WhereT → bool
SelectT → 任意类型
OrderByT → 可比较的 key
GroupByT → 分组 key
FirstOrDefaultT → bool（可选）

4. 链式调用

// 像英语句子一样读
var result = employees
    .Where(n => n.Length > 3)        // 过滤
    .Select(n => n.ToUpper())         // 转换
    .OrderBy(n => n)                  // 排序
    .GroupBy(n => n[0]);              // 分组
顺序很重要： 比如 OrderBy().GroupBy() 和 GroupBy().OrderBy() 结果不同。

5. 延迟执行（先体验下，明天会深入）

var query = numbers.Where(n => n > 2);
// 这一行什么也没做，只是建立"配方"

foreach (int x in query)        // 这里才真正执行过滤
{
    Console.WriteLine(x);
}

6. 常见错误

错对
numbers.where(...).Where(...)
name.lengthname.Length
'hello'（单引号）"hello"（双引号是 string）
.GroupBy().OrderBy()一般先 OrderBy 再 GroupBy

7. 自测

  1. 用 LINQ 把 List<int> { 1, 2, 3, 4 } 过滤为偶数并 × 10，结果？
  2. numbers.First() 和 numbers.FirstOrDefault() 在空集合上的区别？
  3. 给 { "alice", "anna", "bob" } 按首字母分组，几个 group？
查看答案
  1. {20, 40}   — numbers.Where(n => n % 2 == 0).Select(n => n * 10)
  2. First() 抛异常；FirstOrDefault() 返回 0（int 的默认值）
  3. 2 个：'a' → {alice, anna}, 'b' → {bob}
