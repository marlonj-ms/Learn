Lambda 基础

2026-04-17 · 阅读时间约 5 分钟

1. 一句话总结

Lambda = 简洁定义"匿名方法行为"的语法。必须被委托类型（Func / Action / Predicate）承载。

2. 三种形态

// 表达式体（一行，自动 return）
Func<int, int> timesTen = n => n * 10;

// 语句体（多行，必须 return）
Func<int, int> timesTenLog = n =>
{
    Console.WriteLine($"computing {n}");
    return n * 10;
};

// 无返回值（用 Action）
Action<string> print = msg => Console.WriteLine(msg);

3. 参数形态

// 无参数：必须有 ()
Action a = () => Console.WriteLine("hi");

// 一个参数：可省略括号
Func<int, int> b = n => n * 2;
Func<int, int> c = (n) => n * 2;

// 多个参数：必须有括号
Func<int, int, int> d = (a, b) => a + b;

4. Func / Action / Predicate

类型形状用途
Func<T, TResult>T → TResult有返回值
Action<T>T → void无返回值
Predicate<T>T → bool判断真假

5. Lambda 在 LINQ 中的应用

var numbers = new List<int> { 1, 2, 3, 4, 5 };

var evens = numbers.Where(n => n % 2 == 0);     // Func<int, bool>
var tens  = numbers.Select(n => n * 10);        // Func<int, int>

// 复杂例子
var grouped = names
    .Where(n => n.Length > 3)
    .GroupBy(n => n[0])
    .OrderBy(g => g.Key);

6. 闭包（closures）

Lambda 可以"捕获"外面的变量。

int threshold = 3;
var bigOnes = names.Where(n => n.Length > threshold);
//                                       ↑ 捕获外部变量

threshold = 5;     // 此时 bigOnes 还没执行
foreach (var n in bigOnes) ... // 用的是 threshold=5！
闭包 + 延迟执行陷阱： Lambda 捕获的是"变量引用"，不是定义时的"值"。如果外部变量后面变了，lambda 看到的也是新值。

7. 常见陷阱

错原因
var f = n => n * 2;var 无法推断 lambda 类型
numbers.Select(TimesTen(5))这传的是结果，不是方法
numbers.Where(n => n * 2)Where 要 bool，不是 int

  3. Action<int> 和 Func<int> 的区别？
