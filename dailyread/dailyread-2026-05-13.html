Lambda / Delegate / LINQ 串联

2026-05-13 · 阅读时间约 4 分钟

1. 一句话总结

Lambda 写"行为"，Delegate 装"行为"，LINQ 用"行为"。三者是一条链。

2. 三者关系图

你写：  n => n * 10              (lambda 行为)
              │
              ▼
存入：  Func<int, int>           (委托类型 = 容器)
              │
              ▼
传给：  numbers.Select(...)        (LINQ 高阶方法)
              │
              ▼
得到：  IEnumerable<int>          (转换后的集合)

3. Lambda vs 表达式体方法 vs 方法组

// Lambda：匿名行为
n => n * 10

// 表达式体方法（expression-bodied method）：有名字的方法
int TimesTen(int n) => n * 10;

// LINQ 调用
numbers.Select(n => n * 10);     // 用 lambda
numbers.Select(TimesTen);         // 用方法组（method group）
numbers.Select(TimesTen(5));      // ❌ 错：传的是 int 50，不是方法

4. Where vs Select 形状

方法问的问题lambda 返回
Where"我要不要这个元素？"bool
Select"这个元素要变成什么？"任意类型

5. 委托（delegate）是什么

委托 = 能装住方法的类型。
让"方法"变成"可以传递、存储、调用的值"。 
// 内置委托
Func<int, bool> isEven = n => n % 2 == 0;
isEven(4);          // true

// 自定义委托（给名字加意义）
public delegate decimal DiscountRule(decimal price);

DiscountRule tenOff = p => p * 0.9m;
DiscountRule flat20 = p => p - 20m;

decimal Apply(decimal price, DiscountRule rule) => rule(price);

Apply(100m, tenOff);   // 90
Apply(100m, flat20);   // 80

6. 关键认知

  • Lambda 本身没有独立类型，必须被委托类型承载
  • 承载方式：① 显式委托变量 ② 方法参数的类型（如 Where 的参数）
  • var f = n => n + 1; ❌ 不能编译，因为 var 推不出 lambda 的类型
  • Func<int, int> f = n => n + 1; ✅ 显式委托类型

7. 自测

  1. 在 Func<int, bool> isEven = n => n % 2 == 0; 中，isEven 这个变量"装"的是什么？
  2. 为什么 Where(n => n > 3) 中的 lambda 必须返回 bool？
  3. 下面两个等价吗？numbers.Select(TimesTen) vs numbers.Select(n => TimesTen(n))
查看答案
  1. 一个委托实例（delegate instance），它指向一段"接受 int 返回 bool"的方法行为
  2. 因为 Where 用这个 bool 决定"保留还是丢弃"该元素
  3. 等价。第一个是"方法组"，第二个是"包了一层的 lambda"，效果完全相同
