Delegates 深入 + Event 引入

2026-05-15 · 5 分钟复习 · 配套：Delegates-Deep-Dive.cs / Events-Fundamentals.cs

今天主线：把 委托（delegate）当成"能存方法的类型"，然后引出 事件（event）—— 一个受控的 delegate 订阅入口。

1. 类型（type）的本质

类型告诉编译器两件事：

  • 变量能存什么。
  • 变量能做什么操作。
delegate 不是方法本身，而是一个能"指向方法 + 调用方法"的对象。

2. delegate 的定义

public delegate int MathOperation(int a, int b);

这定义了一种委托类型（delegate type）：能存"两个 int 进、一个 int 出"的方法。

MathOperation op = Add;       // 把方法赋给委托变量
int r = op(3, 5);             // 通过委托调用方法

3. 内置三大委托类型

类型形状典型用途
Action<T>T 进，无返回做一件事，没返回
Func<T, TResult>T 进，TResult 出从 T 算出 TResult
Predicate<T>T 进，bool 出判断条件

4. event 为什么需要

把 delegate 字段开放给外部，三种方案对比：

写法问题
public Action<string>? OrderCompleted;太危险：外部能 =null、覆盖、伪造触发
private Action<string>? orderCompleted;太封闭：外部无法订阅
public event Action<string>? OrderCompleted;刚好：只允许 += / -=
event 的本质 = 受控的 delegate。外部只能订阅/取消，触发权留给类自己。
