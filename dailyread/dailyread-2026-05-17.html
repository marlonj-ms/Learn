1. EventHandler<TEventArgs>

public event EventHandler<OrderCompletedEventArgs>? OrderCompleted;
部分含义
EventHandler<OrderCompletedEventArgs>标准事件委托类型（delegate type）
OrderCompleted事件成员（event member）
OrderCompletedEventArgs事件数据类型（event args type）
标准 handler 形状：void Handler(object? sender, OrderCompletedEventArgs e)

2. sender 和 e

OrderCompleted?.Invoke(this, args);
  • this：当前 OrderService 实例，也就是事件发送者（event sender）。
  • args：事件数据对象（event data object），类型是 OrderCompletedEventArgs。
public sealed class OrderCompletedEventArgs : EventArgs
{
    public string OrderId { get; }

    public OrderCompletedEventArgs(string orderId)
    {
        OrderId = orderId;
    }
}

3. 事件订阅关系

orderService.OrderCompleted += emailNotifier.HandleOrderCompleted;
部分含义
OrderCompleted事件（event）
HandleOrderCompleted事件处理方法（event handler method）
+=把 handler method 注册到事件调用列表中
HandleOrderCompleted 不是新 event。它是被订阅到 event 上的 handler method。

4. OnOrderCompleted

protected virtual void OnOrderCompleted(OrderCompletedEventArgs e)
{
    OrderCompleted?.Invoke(this, e);
}

生产代码常把业务动作和事件触发分开：

CompleteOrder(...)      // 业务动作（business action）
OnOrderCompleted(...)  // 事件触发方法（event raising method）
如果类可继承，protected virtual 让子类可以在事件触发前后加自己的逻辑。

5. 继承与多态

public class PriorityOrderService : OrderService
{
}
  • OrderService：基类（base class）/ 父类（parent class）
  • PriorityOrderService：派生类（derived class）/ 子类（child class）
  • : OrderService：继承自 OrderService
OrderService service = new PriorityOrderService();
  • variable type = OrderService
  • real object type = PriorityOrderService
variable type decides what members you can call. real object type decides which overridden method actually runs.

6. virtual / override / sealed / base

关键字一句话
virtual父类允许子类重写这个方法
override子类提供自己的方法实现
sealed class整个类不能再被继承
sealed override这个方法不能被更下层子类继续重写
base.SomeMethod()调用父类版本的方法
 
protected override void OnOrderCompleted(OrderCompletedEventArgs e)
{
    Console.WriteLine("Book-specific logic before event");
    base.OnOrderCompleted(e);
}
base 不会创建新的父类对象。它表示访问当前对象里的父类行为。
