Event 流程图 + Async/Await 入门

2026-05-18 · 5 分钟复习 · 配套：Good-Design / Async-Demo / Event-Flow-Reference.md

今天主线：用 Bad/Good/Your 三种设计对比 event 标准模式，然后进入 异步等待（async/await）的心智模型 + 顺序 vs 并行运行。

1. Event 标准模式（生产推荐）

public event EventHandler<OrderCompletedEventArgs>? OrderCompleted;

public void CompleteOrder(string orderId)
{
    var args = new OrderCompletedEventArgs(orderId);
    OnOrderCompleted(args);
}

protected virtual void OnOrderCompleted(OrderCompletedEventArgs e)
{
    OrderCompleted?.Invoke(this, e);
}
所有触发逻辑集中在 OnXxx，业务方法只构造 EventArgs 然后调它。子类可 override 注入额外行为。

2. Event 调用流程

业务方法
   ↓ 构造 EventArgs
   ↓ 调 OnXxx(e)
       ↓ 调 event?.Invoke(this, e)
           ↓ 同步逐个调订阅者
   ↓ 全部跑完后业务方法继续
事件调用列表（invocation list）同步执行。一个 handler 抛异常，后面的 handler 可能不跑。

3. EventArgs 应该不可变（immutable）

public sealed class OrderCompletedEventArgs : EventArgs
{
    public string OrderId { get; }                      // 只读属性
    public OrderCompletedEventArgs(string orderId) { OrderId = orderId; }
}

4. Async/Await 三大要点

关键字含义
async声明这个方法可能包含 await，编译器把它编成状态机
Task / Task<T>"未来才会有的结果"
await"在这里挂起，等结果出来再继续，期间线程可以做别的事"

5. 顺序 vs 并行（demo 重点）

// 顺序：A 完了才开始 B，慢
var a = await LoadAsync("A");
var b = await LoadAsync("B");

// 并行：A 和 B 同时开始，快
var taskA = LoadAsync("A");
var taskB = LoadAsync("B");
await Task.WhenAll(taskA, taskB);
口诀："启动多个 Task，最后再 WhenAll"。这是 .NET 异步编程的肌肉记忆。
