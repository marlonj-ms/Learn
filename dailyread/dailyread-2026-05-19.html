2. 初始化器三兄弟

// 集合初始化器（collection initializer）
var nums = new List<int> { 1, 2, 3 };
// 编译器翻译成：new List() + Add(1) + Add(2) + Add(3)

// 对象初始化器（object initializer）
var book = new Book { Title = "X", Pages = 100 };
// 需要 public 无参构造 + public 可写属性

// 索引初始化器（index initializer）
var dict = new Dictionary<string, int> { ["a"] = 1, ["b"] = 2 };
集合初始化器三个条件：实现 IEnumerable + 有 Add 方法 + 能 new 出实例（无参构造或显式传参）。

3. LINQ 链路：管道思维

booklist
    .Where(b => b.Pages > 500)        // 过滤（filter）
    .Select(b => b.Title)             // 投影（projection）
    .OrderBy(t => t)                  // 排序，返回新序列
    .Distinct()                       // 去重
// 后续用 foreach 或 ToList() 真正执行
LINQ 链不放副作用。Select 是数据变换，不要在里面 Console.WriteLine。打印放在 foreach 里。

4. List.Sort + Comparison<T>

booklist.Sort((a, b) => a.Pages - b.Pages);   // 升序
//              ↑ 返回 int 不是 bool
返回值含义
负数a 排前
0相等
正数b 排前
OrderBy（LINQ）返回新序列，原 list 不变。List.Sort 原地排序，会改原 list。

5. Dictionary 计数模式（生产首选）

var counts = new Dictionary<string, int>();
foreach (var word in words)
{
    counts.TryGetValue(word, out int current);   // 不在时 current = 0
    counts[word] = current + 1;                  // 在则改不在则插
}
TryGetValue 只读，不会插入。真正插入的是后面那行索引器赋值。
out int current 在 key 不存在时被赋为 int 的默认值 0，所以省掉 if/else。

6. 反复踩的坑（今天）

  1. 私有字段只能类内访问。Main 里写 _title 错了 3 次。外部必须走 public 属性。
  2. 插值字符串少 $。"{x}" 打字面，$"{x}" 才替换。
  3. LINQ 里别打印。Where/Select 是管道，副作用放外面。
  4. Sort 比较器返回 int，不是 bool。
  5. dict 三种写法：Add（在则抛）、索引器赋值（在则改不在则插）、TryAdd（在则不动返回 false）。

8. Doorbell → TemperatureSensor：事件模式打通

事件的最短心智模型：event 是 handler 方法列表；+= 添加监听者，-= 移除监听者，?.Invoke 安全通知所有监听者。
Doorbell 迷你练习TemperatureSensor 正式练习
DoorbellEventArgsThresholdExceededEventArgs
PresserNameReading / Threshold / Timestamp
Press(string name)RecordReading(double celsius)
RangThresholdExceeded
OnRang(...)OnThresholdExceeded(...)
Person.OnDoorbellRangBuzzer / Logger.OnThresholdExceeded
 
protected virtual void OnThresholdExceeded(ThresholdExceededEventArgs e)
{
    ThresholdExceeded?.Invoke(this, e);
}
protected virtual OnXxx 是发布者内部的事件触发入口。外部代码不能随便触发事件；子类如果需要，可以 override 这个步骤。

9. override + base.OnXxx 的图

ndb.Press("ndb1")
  -> Doorbell.Press(...)
  -> OnRang(args)
  -> 真实对象是 NoisyDoorbell，所以进入 NoisyDoorbell.OnRang(args)
  -> base.OnRang(args)
  -> Doorbell.OnRang(args)
  -> Rang?.Invoke(this, args)
  -> subscribers run
override = 子类接管这个 virtual 步骤；base.OnXxx(e) = 子类做完额外行为后，回到父类默认流程继续通知订阅者。

10. 今天事件练习的关键纠错

  1. Press() 不一定需要参数。方法只有在需要 caller 提供数据时才需要参数。
  2. 有自定义事件数据时，三处类型必须一致：EventArgs 子类、EventHandler<T>、handler 的第二个参数 T e。
  3. Press 里如果直接 Rang?.Invoke(...)，会绕过 OnRang，子类 override 不会执行。
  4. Buzzer 和 Logger 不是 TemperatureSensor 的子类；它们是 subscriber/listener。
  5. public double Threshold { get; } 是 get-only property，外部不能改；public double Threshold; 是 public field，外部可以改。

11. Unit Test 下一步：从 0 开始

不要直接跳 xUnit。先做 console 版手动测试：

// Arrange
var sensor = new TemperatureSensor(100.0);
bool eventFired = false;

sensor.ThresholdExceeded += (sender, e) =>
{
    eventFired = true;
};

// Act
sensor.RecordReading(101);

// Assert
Console.WriteLine(eventFired ? "PASS" : "FAIL");
下一节学习顺序：先理解 Arrange / Act / Assert，再理解 [Fact] 是 xUnit 给 test runner 的标签，最后用 Assert.True(...) 替换手写 PASS/FAIL。
