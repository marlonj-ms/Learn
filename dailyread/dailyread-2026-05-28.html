嵌套泛型阅读流利度 🧠

2026-05-28 · 6 级阶梯 · 配套：完整 Summary · 可跑 demo

一句话：从 List<int> 一路爬到 Func<Dictionary<string,List<int>>, List<string>>，6 级阶梯一气通关。最大收益是口诀 #43 ——读 Func<...> 时目光直接弹到最后一个泛型参数 = 返回类型，前面全是噪音。这一条单独就抵 LINQ 半本书。

6 级阶梯地图

级别签名关键点
L1List<int>3 种初始化语法等价；空 List ≠ null
L2List<T>开放泛型 vs 封闭泛型；T 没替换前不能 new
L3List<List<int>> / Dictionary<string,int>锯齿 vs 矩形；[] 两种语义（index vs key）
L4List<List<List<int>>>剥洋葱模式不变，只是多剥一层
L5Dictionary<string, List<Order>>业务签名 + 链式取值类型追踪
L6Func<Dictionary<string,List<int>>, List<string>>delegate 包嵌套泛型；最后一参数 = 返回类型

核心心法 — 剥洋葱解码法（3 步）

Step 1 — 找最外层

看最左的类型名 + 它紧跟的 <...>，这就是最外面的盒子。

Step 2 — 数 < > 配对深度，切顶层逗号

只有深度为 0 时遇到的逗号才属于最外层，嵌套类型自己的逗号要忽略。

Func<Dictionary<string, List<int>>, List<string>>
     ^                            ^               ^
     深度 1                       深度 0          深度 0
                                  ↑ Func 的分界    Func 收尾

Step 3 — 每层翻译成业务白话

  • Dictionary<string, List<int>> → "按 string（如学生名）查 List<int>（这个学生的分数列表）"
  • List<string> → "字符串列表（如学霸名单）"
  • 串起来："喂一张分数总表，吐一张学霸名单"

关键代码片段

1. 3 种 List 初始化语法（等价）

var a = new List<int>() { 10, 20, 30 };   // 完整
var b = new List<int>  { 10, 20, 30 };    // 省略空括号
List<int> c = new()    { 10, 20, 30 };    // 目标类型 new() (C# 9+)

2. Dictionary 安全取值

// ❌ 缺 key 抛 KeyNotFoundException
var x = dict["maybe-missing"];

// ✅ TryGetValue
if (dict.TryGetValue("k", out var v)) Console.WriteLine(v);

// ✅ 忽略大小写
var safe = new Dictionary<string, int>(StringComparer.OrdinalIgnoreCase);

3. 链式取值类型追踪

Dictionary<string, List<Order>> ordersByCustomer = ...;

ordersByCustomer            // Dictionary<string, List<Order>>
ordersByCustomer["alice"]   // List<Order>
ordersByCustomer["alice"][0] // Order
ordersByCustomer["alice"][0].Total // decimal

// 左边变量类型 = 链式最后一步的类型
decimal firstTotal = ordersByCustomer["alice"][0].Total;

4. Func 解码法

Func<A, B, C> f;
//   ^^ ^   ^
//   输入   返回（最后永远是返回）

Func<List<Order>, Dictionary<string, decimal>> summarize;
//   ^─ 输入 ─^  ^──────── 返回 ──────────^

反复踩的坑（追加到 cheatsheet 54–62）

坑 54：把空 List 当 null —— list == null 永远 False，要用 list.Count == 0
坑 55：?. 短路 —— 链中任一段 null，后面全跳过返回 null
坑 56：List<List<T>> 和 T[,] 混用 —— 不规则走锯齿；固定矩形（棋盘 / 图像）走 T[,]
坑 57：嵌套下标越界：写 m[Count][Count] —— 改 m[^1][^1] 或 m[Count-1][Count-1]
坑 58：dict["missingKey"] 抛 KeyNotFoundException —— 用 TryGetValue
坑 59：dict["Alice"] ≠ dict["alice"] —— 默认 case-sensitive，要忽略传 StringComparer.OrdinalIgnoreCase
坑 60：decimal d = 1000.0; 编译错 —— double 字面量赋 decimal；加后缀 m：decimal d = 1000.0m;
坑 61：读 Func<...> 时眼睛跑去看"中间最显眼的东西"忘了找返回类型 —— 第一眼直接定位最后一个 , 后面那块
坑 62：嵌套泛型逗号有两种身份 —— 数 < > 配对深度 = 0 时的逗号才属于外层
