LINQ 链式流利度 🔗

2026-05-29 · 3 Parts + 毕业题 · 配套：完整 Summary · Part 1 详细笔记 · 可跑 demo

一句话：「Where 是配方，Select 换形状，GroupBy 装袋子，OrderBy 升级成 IOrderedEnumerable<> 等 ThenBy，ToList() 才下锅」—— 一条链的每一步返回类型 + 延迟时机全都盲读得出。毕业题 20.5/21 通过。

3 大签名解码

方法签名关键
WhereIEnumerable<TSource> Where<TSource>(this IEnumerable<TSource>, Func<TSource,bool>)1 个泛型参，保类型，减个数
SelectIEnumerable<TResult> Select<TSource,TResult>(this IEnumerable<TSource>, Func<TSource,TResult>)2 个泛型参，换类型，保个数
GroupByIEnumerable<IGrouping<TKey,TSource>> GroupBy<TSource,TKey>(this IEnumerable<TSource>, Func<TSource,TKey>)双层嵌套登场
OrderByIOrderedEnumerable<TSource> OrderBy<TSource,TKey>(this IEnumerable<TSource>, Func<TSource,TKey>)升级版接口，才能链 ThenBy

IGrouping<TKey,TElement> 那个袋子

public interface IGrouping<out TKey, out TElement> : IEnumerable<TElement>
{
    TKey Key { get; }   // 袋子标签
}
  • 外面贴 Key 标签
  • 本身 : IEnumerable<TElement> → 袋子可以直接 foreach
  • 双层泛型 = 双层 foreach
foreach (var g in orders.GroupBy(o => o.Status))
{
    Console.WriteLine($"标签: {g.Key}");
    foreach (var o in g)          // ← 袋子本身可 foreach
        Console.WriteLine($"  {o.Id}, Status={o.Status}");  // Status 没丢!
}

口诀 — 8 个新条目（#44–51）+ 1 个意外收获（#52）

#口诀
44LINQ 操作返回 IEnumerable<T>，不是你输入的具体类型。能 foreach ≠ 是 List
45IEnumerable<T> 每被 foreach/Count/Any 一次，整本配方重跑一次。要复用先 .ToList()
46延迟执行 = 配方在 ToList()/foreach 那一刻才看数据源。期间改源会看到新值
47Select 保个数换类型；Where 保类型减个数
48new(...) 目标类型 new；[...] 集合表达式；都靠左边推断。var + 这两种 = 错
49GroupBy 返回 IEnumerable<IGrouping<TKey,T>> 双层。外层一串袋子，每个袋子带 Key 且可 foreach
50GroupBy 袋子顺序在 LINQ to Objects 是"首次出现"序，但接口不承诺。要顺序就 OrderBy
51GroupBy(keySelector) 只贴标签，不改元素。袋里装的是完整 TSource，包括分组字段本身
52一个 assembly 只能有一个入口点。顶级语句存在时编译器优先选它，Main() 被降级成普通静态方法（仅 warning CS7022）

💣 LINQ 最经典生产事故 — Multiple Enumeration

反例（数据库被打 3N 次）： 
var query = users
    .Where(u => SlowDatabaseLookup(u.Id))   // N 次往返
    .Select(u => u.Name);

if (query.Any())                                // N 次
{
    Console.WriteLine($"找到 {query.Count()} 个");   // 又 N 次
    foreach (var name in query) ...             // 再 N 次
}
正例（加一行，N 次）： 
var names = users
    .Where(u => SlowDatabaseLookup(u.Id))
    .Select(u => u.Name)
    .ToList();                                  // ← 一次煮熟

if (names.Count > 0)
{
    Console.WriteLine($"找到 {names.Count} 个");
    foreach (var name in names) ...
}
ReSharper / Rider 警告："Possible multiple enumeration of IEnumerable"

🎓 毕业题：Top Customer 综合链

var report = orders
    .Where(o => o.Status == "Paid")
    .GroupBy(o => o.CustomerName)
    .Select(g => new {
        Customer = g.Key,
        OrderCount = g.Count(),
        TotalSpent = g.Sum(o => o.Amount)
    })
    .OrderByDescending(x => x.TotalSpent)
    .ToList();

💀 今天反复踩的坑（追加到 cheatsheet 63–67）

坑 63：多次枚举 IEnumerable<T> —— 每 foreach/Count/Any 一次配方重跑一次。复用前必须 .ToList() 物化
坑 64：把 Where() 的结果当 List<T> —— 输入 List<int>，输出还是 IEnumerable<int>（具体是 *Iterator）
坑 65：var + 集合表达式 [...] 或目标类型 new(...) —— 编译失败，因为右边没类型可推断，必须左边写明类型
坑 66：假设 GroupBy 袋子顺序固定 —— LINQ to Objects 给"首次出现"序，但接口不承诺；其它 provider（EF、PLINQ）不保证。要顺序就显式 OrderBy(g => g.Key)
坑 67：同一 assembly 顶级语句 + Main() 共存 —— 编译器静默选顶级语句，你写的 Main() 被降级（仅 warning CS7022），运行结果可能"看似正常实则跑错入口"。解决：删并行 Program.cs，或在 csproj 加 <StartupObject>Namespace.Program</StartupObject> 强制报错
