DbContext — 一次数据库会话的总管

  • 继承 EF Core 的 DbContext 基类
  • 内部装着 DbSet<T>，每个 = 一张表入口
  • 三件大事：连接管理 / 变更追踪 / 一次性提交 SaveChanges()
  • OnConfiguring → 告诉 EF 连哪个库（本 demo 用 SQLite 本地文件）

demo 输出 3 大看点

1. 建表 — 没写一行 SQL

CREATE TABLE "Readings" (
    "Id" INTEGER NOT NULL CONSTRAINT "PK_Readings" PRIMARY KEY AUTOINCREMENT,
    "Celsius" TEXT NOT NULL );
  • Id 约定自动成主键 + 自增
  • 表名 Readings（复数 = DbSet 属性名）
  • decimal 在 SQLite 存成 TEXT（无原生 decimal，文本保精度）

2. 插入 — 参数化 INSERT，天然防 SQL 注入

Executed DbCommand [Parameters=[@p0='22.5']]
INSERT INTO "Readings" ("Celsius") VALUES (@p0)

3. 查询 — 高光时刻 🌟

db.Readings.Where(r => r.Celsius > 30m).OrderByDescending(r => r.Celsius)
// 翻译成：
SELECT "r"."Id", "r"."Celsius" FROM "Readings" AS "r"
WHERE ef_compare("r"."Celsius", '30.0') > 0
ORDER BY "r"."Celsius" ... DESC
只回 2 行（35.5 / 31.0）；18.0 / 22.5 根本没离开数据库 = 远程过滤省带宽。

口诀 — 5 个新条目（#74–#78）

#口诀
74DbSet<T> : IQueryable<T> —— 这一行继承关系，就是 LINQ 能变成 SQL 的全部秘密。
75DbSet<T> 是 EF 在 DLL 里定义的表入口类；Set<T>() 是 DbContext 基类上「把表入口取出来」的方法。
76类 = 行的模板，DbSet<类> = 整张表。（DbSet<Reading> 之于 Reading ≈ List<Reading> 之于 Reading）
77AsEnumerable() 是一道「翻译截止线」 —— 线之前能进 SQL，线之后全在内存。误用 = 全表拉回 + 内存过滤。
78EF 靠「快照对比（snapshot comparison）」知道你改了什么 —— 查询时拍快照，SaveChanges 时逐字段 diff（明天验证）。
